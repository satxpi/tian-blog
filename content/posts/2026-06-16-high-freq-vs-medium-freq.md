## 一个常见的误解

说到量化交易，很多人脑子里浮现的画面是：服务器机房、毫秒延迟、高速专线、算法每秒执行上千笔交易。

这确实是高频交易（HFT）的样子，但这不是普通人能做的事。

高频交易的门槛是**基础设施**，不是算法。顶级 HFT 公司每年在服务器托管、专线网络、硬件上花的钱以百万计。没有这个基础设施，再聪明的算法也没用——因为你的订单总是比别人慢，别人早就把价格吃掉了。

对个人量化来说，**中低频策略才是正确的赛道**。

---

## 频率分类

| 频率 | 持仓周期 | 典型换手率 | 核心优势来源 |
|------|----------|------------|-------------|
| 超高频（μHFT） | 毫秒~秒 | 每秒数百次 | 延迟优势、订单簿套利 |
| 高频（HFT） | 秒~分钟 | 每天数十次 | 市场微结构、做市商优势 |
| 日内（Intraday） | 分钟~小时 | 每天 1-10 次 | 短期价格规律、盘中波动 |
| 低频（Low） | 天~周 | 每周 1-5 次 | 基本面、情绪因子 |
| 中低频（Mid-Low） | 周~月 | 每月几次 | 价值回归、趋势跟踪 |

A 股的 T+1 机制决定了真正的日内高频（买了当天卖）只适合卖出操作，所以实际上大多数个人量化在做**低频到中低频**。

---

## 高频交易的真实门槛

国内高频交易圈有个说法：**高频的护城河是微秒，不是算法**。

具体来说：

**网络延迟**：交易所距离服务器越近越好。上海证券交易所在浦东，深圳交易所在前海。托管在交易所附近机房的服务器，比你家里的电脑延迟低 10 倍以上。

**技术栈**：高频策略用 C++ 或者 Rust 写，Python 的速度完全不够。一些极端的场景用 FPGA（现场可编程门阵列）直接硬件处理数据。

**数据源**：高频策略依赖 Level 2 逐笔成交数据、委托队列数据，这类数据每个月几千到几万。日线数据对高频毫无意义。

**资金规模**：高频策略每笔收益可能只有 0.01%，需要大资金量才能绝对金额可观。同时交易所对单账户的交易频率也有限制。

## 个人量化的实际选择

去掉高频，个人量化的选择范围变得清晰多了。

### 日线低频（持仓数天到数周）

**核心逻辑**：基于因子模型预测未来收益，定期调仓。

典型策略：
- **多因子选股**：每周或每月根据因子排名调仓一次
- **趋势跟踪**：买入上升趋势的股票，止损下降趋势的
- **均值回归**：超跌的质地好的股票会反弹

```python
# 简单的每周调仓多因子策略
def weekly_rebalance(factor_scores: pd.DataFrame, top_n: int = 50) -> list:
    """根据因子分数选出 Top N 股票"""
    today_scores = factor_scores.iloc[-1].sort_values(ascending=False)
    return today_scores.head(top_n).index.tolist()
```

**优势**：
- 交易成本低（换手率低）
- 对执行基础设施要求低，普通券商 API 就够
- 可以用 Python 完整实现

**劣势**：
- 容量有限，资金大了会影响价格（对个人影响不大）
- 信号衰减快，因子有效性随时间变化

### 分钟级中频（持仓数小时）

A 股可以当天卖出昨天买的票，所以中频日内策略是可行的。

典型策略：
- **开盘 Gap 策略**：利用开盘价跳空后的价格回归
- **板块轮动日内版**：追板块热点的日内版本
- **量价异动**：成交量突然放大时的短期跟踪

```python
# Gap 策略：开盘跳空后追涨
def gap_strategy(open_price, prev_close, volume, avg_volume):
    gap_pct = (open_price - prev_close) / prev_close
    volume_ratio = volume / avg_volume
    
    # 跳空 > 2%，量比 > 3 倍，追涨
    if gap_pct > 0.02 and volume_ratio > 3:
        return "buy"
    # 跳空 < -2%，放量，做空（融券）
    elif gap_pct < -0.02 and volume_ratio > 3:
        return "short"
    return "hold"
```

**优势**：盘中信号及时，波动捕捉能力强

**劣势**：
- 需要分钟级数据（一般要付费）
- 交易频率高，手续费压力大
- 盯盘压力大（或者需要全自动化执行）

## 自动化执行

中低频策略手动执行太痛苦，需要自动化。国内券商大多数支持量化接口：

### 东方财富 easyquant

```python
from easyquant import QuotationService, TradeAPI

# 连接行情
quot = QuotationService()
quot.subscribe(["000001", "600000"])

# 交易
trade = TradeAPI(account="xxx", password="xxx")
trade.buy("000001", price=15.5, volume=100)
trade.sell("000001", price=16.0, volume=100)
```

### 同花顺 iFinD

```python
# 通过 HTTP API 发单（需申请量化账户）
import requests
headers = {"Authorization": "Bearer your_token"}
order = {"symbol": "000001", "side": "buy", "qty": 100, "price": 15.5}
requests.post("https://api.10jqka.com.cn/order", json=order, headers=headers)
```

### 万得 API（Wind Python API）

主要用于数据获取，但也支持部分交易操作：

```python
from WindPy import w
w.start()
data = w.wsd("000001.SZ", "close,volume", "2025-01-01", "2026-06-01")
```

## 中低频策略的核心挑战

**1. 信号时效性**

日线因子的有效期一般是几天到几周。市场风格切换时，原来有效的因子会突然失效。需要持续监控信号质量。

**2. 市场容量**

A 股日成交量 5000-8000 亿，中小市值股票单票日成交量可能只有几千万。个人资金量不大时没问题，但资金量到了几百万以上，频繁操作会影响价格。

**3. 风格漂移**

量化模型倾向于挖掘历史上有效的模式。问题是市场风格会变——2021 年有效的小市值因子，在 2023 年的央国企行情里完全没用。

## 给个人量化的建议

综合来看，对于大多数个人量化：

1. **从低频开始**：先把因子选股、回测框架、自动化执行搞通，再考虑提频
2. **不要碰高频**：基础设施门槛够不到，做了也是亏钱
3. **重视风控**：中低频策略每次调仓时间充足，有机会做精细的仓位管理
4. **量力而行**：日线低频策略对硬件要求低、可以白天正常工作，合理的起点

资金少的时候，量化的意义主要是**学习和验证思路**，不是追求绝对收益。等想清楚了再慢慢加。
