二月 AI 圈又多了个新东西。

Nous Research 发了一个叫 **Hermes Agent** 的开源智能体。

一句 curl 装好，跑在你自己的机器上，MIT 协议完全开源。

但这不是重点——重点是它的核心设计：**会记住你。**

## "记住你"是什么意思

传统 AI：每次对话从零开始。你告诉他"我在做一个电商项目，用的是 Vue 3 + Node.js"，他听懂了。然后你关了对话、明天再打开——全忘了。你得重新说一遍。

Hermes Agent：你第一次说了你的技术栈，它记住了。以后每次对话，不用再说。

更狠的是：你让它修一个 bug，它折腾了半天终于修好了。它把修这个 bug 的过程写成一个"技能"存起来。下次遇到类似的 bug，直接调用这个技能，一分钟搞定。

这叫**闭环学习**——AI 不是每次从零开始，而是越用越厉害。

## 实际体验

我在 WSL2 里装了一个，连了 Telegram。

第一周没太大感觉——就是个普通的 AI 助手，回答问题、帮忙写代码。

第二周开始不一样了。

我让它查一个 AWS 账单问题。它翻了半天文档，找到了原因是某个 S3 bucket 的 lifecycle policy 没配。然后它自动创建了一个技能：`aws-cost-check`，记录了怎么查账单、常见问题和解决方法。

第三天，我问："S3 又在乱收费了，帮我查一下。"

它直接调用了 `aws-cost-check` 技能，三分钟给了结论："不是 S3，是 Lambda 的并发配置改了。"

**它不需要我重新教一遍怎么查 AWS 账单。它记住了。**

## Hermes 跟 OpenClaw 的区别

这是很多人问的问题。两个都是开源的、自托管的、MIT 协议的 Agent，到底有什么不同？

| | OpenClaw | Hermes Agent |
|---|----------|-------------|
| 发布时间 | 2026.01 | 2026.02 |
| 重点 | **操作能力** | **记忆能力** |
| 连接平台 | 15+ | 5（Telegram/Discord/Slack/WhatsApp/Signal） |
| 技能系统 | 手动/市场安装 | 自动创建 |
| 记忆 | 靠 prompt 注入 | 持久化存储 |
| 适合 | 需要操作很多系统的人 | 需要AI记住上下文的人 |
| 生态 | ClawHub 技能市场 | agentskills.io 开放标准 |

一句话区别：OpenClaw 偏向"替你操作"，Hermes 偏向"替你记住"。

当然，两者都在快速发展。OpenClaw 后面肯定会加强记忆，Hermes 也会扩展平台连接。

## 一个隐忧

Hermes 把关于你的一切都存在本地——你的项目、你的偏好、你的代码、你的工作方式。

这很安全（数据不离开你的机器）。

但也很危险——如果哪天你的硬盘坏了，或者你换了电脑，"记忆"就丢失了。

我提交了一个 feature request：能不能加一个加密的云端备份选项？团队回复说在 roadmap 上。

在加之前，自己记得备份 `~/.hermes/` 目录。

## 我的结论

如果你需要一个能记住你所有项目、习惯、偏好的 AI 助手——Hermes 是最佳选择。

如果你更需要一个能操作多个平台的执行型 Agent——OpenClaw 更好。

当然，最好的方案是两个都装。它们不冲突。

反正都是免费的。
