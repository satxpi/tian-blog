三月第二个周末，我的朋友圈被"养虾"刷屏了。

不是真的养虾。是**OpenClaw**——那个一月份发布的开源 Agent 框架。

腾讯云推出了 OpenClaw 的"一键部署"方案：在你已有的 Lighthouse 轻量服务器上，点一下按钮，三分钟装好一个完整的 AI Agent。不需要配环境、不需要改配置、不需要看文档。

这彻底改变了 OpenClaw 的用户画像。

## 一键部署之前 vs 之后

**之前：**

OpenClaw 的用户几乎全是开发者。要装 Docker、要配 API key、要改一堆配置——没有编程基础的人根本搞不定。

用户在 GitHub 上提 issue，最多的不是功能建议，是"装不上怎么办"。

**之后：**

一个完全不会写代码的人，花 30 秒在腾讯云控制台点一下"部署"，三分钟后就有一个运行中的 AI Agent。加上飞书/企业微信的通道，手机发消息就能用。

我周围有三个不做技术的人装了——一个做设计的、一个做自媒体的、一个做跨境电商的。

这不是"技术扩散"，这是**门槛坍塌**。

## 为什么腾讯这一手很聪明

腾讯不是随便推广一个开源项目。他们看明白了三件事：

**第一，Agent 需要落地场景。** OpenClaw 的"手机上指挥 AI 干活"这件事，比任何 Agent 产品都更接地气。因为它不改变你的工作流——你本来就在用微信/飞书，多加一个"AI 联系人"就行了。

**第二，中国用户更需要"自有 Agent"。** 中国用户对 OpenAI/Google 的云端服务天然不信任（访问慢、数据在境外、可能被墙）。能跑在自己的服务器上这件事，在中国是刚需。

**第三，腾讯有场景、有渠道、有用户。** 企业微信、飞书、腾讯云——中国 90% 的公司都在用腾讯系的至少一个产品。OpenClaw 一键接入企业微信，这不是功能，是**分发渠道**。

## 小龙虾出圈的真实数据

腾讯云公布的数据：

- 一键部署方案上线第一周：12 万次部署
- 接入企业微信的组织：超过 8000 家
- 最常见的三个使用场景：周报自动生成、客服自动回复、数据分析简报

一个开源的、社区维护的项目，突然有了十几万用户——这对 OpenClaw 团队来说是幸福的烦恼。

幸福：用户多了，生态活了。ClawHub（技能市场）上的技能从几十个涨到了一千多个。

烦恼：反馈量爆炸，issue 列表从两百多涨到一千多，社区管理员不够用了。

## 但也带来了新问题

我之前说 OpenClaw 的好处是"数据在你手里"。但这有个前提——**你知道怎么管理服务器**。

一键部署让很多不懂运维的人开始"养虾"。结果是什么？暴露在公网上的 Agent、默认密码没改、API key 泄露。

腾讯后来加了个"安全配置检查"功能，但说实话——让一个不懂技术的人管一个有 root 权限的 AI Agent，这本身就是安全隐患。

这不是 OpenClaw 的问题，这是 AI Agent 普及过程中必然要面对的问题。就像智能手机普及后每个人都得学会设密码一样。"养虾"也得学会基本的安全操作。

## 整个 AI Agent 的格局变了

OpenClaw + 腾讯一键部署这件事，打破了一个格局：

之前做 Agent 的思路是"我们做一个超级牛逼的 Agent 产品，你们来用"。OpenAI、Google、Meta 都是这个思路。

OpenClaw 走的是另一个方向："我们提供一个 Agent 框架，你们自己往上加东西，想怎么用就怎么用。"

而腾讯一键部署，又往前推了一步——**"我们帮你把 Agent 装好，你只管用"。**

这跟 2008 年 iPhone 的 App Store 有点像。

不是苹果发明了所有 App——是苹果搭了个台子，别人来唱戏。

OpenClaw 就是 AI Agent 世界的 App Store。
