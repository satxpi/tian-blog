## 序

前几天跟一个做投资的朋友吃饭，他问了我一个问题。

"老田，你说现在这些 AI Agent，到底哪个是真的能替我干活的？"

我说你具体想让它干什么。

他说："帮我查资料、写报告、做表格、回邮件。最好是我不说它也能猜到我想要什么。"

我笑了。他不是在问 AI。他是在问一个**不要工资的助理。**

但这个问题确实值得认真回答。因为从 2025 年初到现在这一年半，AI 行业的变化速度，快到了即使我这种每天都关注的人也经常跟不上。

2025 年 1 月，我们还在讨论"AI 能不能写出可用的代码"。2026 年 6 月的今天，AI 已经在替开发者创建 GitHub Pull Request、自动修 Bug、自己写自己下一个版本的训练代码了。

这篇文章不聊概念，不画饼。就把这一年半发生的事、市面上最火的几个 Agent、它们到底能不能用——掰开了聊。

---

## 一、16 个月，世界变了三次

我拉了一条时间线，从 2025 年 1 月到 2026 年 6 月。你会发现，每隔三四个月，就有一个节点彻底改变我们对 AI 的认知。

### 第一阶段：推理觉醒（2025 年 1 月 — 3 月）

这个阶段的主题是"AI 开始会思考了"。

- **2025 年 1 月**：DeepSeek 开源 R1 推理模型。震惊行业——不是因为它比 OpenAI 强，而是它用极低的训练成本达到了接近 o1 的水平。一夜之间，国产模型有了真正的存在感。同月，OpenAI 发布 Operator（研究预览），第一个能操控浏览器的 AI 智能体出现。
- **2025 年 2 月**：Anthropic 发布 Claude 3.7 Thinking，首次推出"扩展思考"模式。xAI 的 Grok 3 上线，10 倍于前代的算力训练。
- **2025 年 3 月**：Gemini 2.5 Pro 实验版发布，100 万 token 上下文——你能把一整本《三体》扔进去让它分析了。

**这个阶段的共识**：AI 不再是"预测下一个词"，而是真正在**推理**。但推理的可靠性还很不稳定——你永远不知道它什么时候会在严肃结论后面接一句胡话。

### 第二阶段：模型井喷（2025 年 4 月 — 7 月）

模型数量爆炸，大厂和开源社区全部下场。

- **2025 年 4 月**：Meta 发布 Llama 4，OpenAI 发布 GPT-4.1（100 万上下文）。开源社区沸腾——Qwen 3 系列发布，阿里一口气出了多个尺寸。
- **2025 年 5 月**：**关键节点**——Anthropic 发布 **Claude 4 Opus**，支持 7 小时自主运行模式。同月 OpenAI 发布 **Codex**（基于 o3），第一个真正的自主编程 Agent，可以写代码、调试、创建 GitHub PR。
- **2025 年 7 月**：OpenAI 的模型在 IMO 2025（国际数学奥林匹克）拿到金牌。xAI 发布 Grok 4，ARC-AGI 基准得分创新高。

**这个阶段的意义**：AI 正式从"工具"变成了"同事"。Claude 4 Opus 的 7 小时自主运行和 Codex 的自动编程，标志着 AI 具备了**持续工作**的能力，而不仅仅是"一问一答"。

### 第三阶段：Agent 元年（2025 年 8 月 — 12 月）

下半年，Agent 彻底火了。

- **2025 年 8 月**：OpenAI 发布 **GPT-5**——编程能力质的飞跃，动态思考模式减少幻觉。Google 推出 Gemini 2.5 Deep Think，可以"思考更久再回答"。
- **2025 年 9 月**：Qwen-3-Max（超万亿参数）和 Suno v5（录音室级 AI 音乐）发布。AI 覆盖的领域从文字扩展到音乐和视频。
- **2025 年 11 月**：OpenAI 发布 GPT 5.1 系列（含 Thinking 和 Instant 模式）。Google Gemini 3.0 登场。Anthropic 发布 Claude Opus 4.5。
- **2025 年 12 月**：GPT-5.2 发布，内置 Codex Agent。**Manus 被 Meta 以 20 亿美元收购**——一个中国团队做的通用 Agent，被天价收购，引发行业震动。

**这个阶段的特点是"寒武纪大爆发"**：Agent 不再只是概念，所有大厂都在推自己的 Agent 产品。但问题也来了——谁能用？谁好用？谁在割韭菜？市场开始分化。

### 第四阶段：分化与务实（2026 年 1 月 — 6 月）

进入 2026 年，热潮退去，真相浮出水面。

- **2026 年 1 月**：Anthropic 推出 **Claude Cowork**——可以跨文档、表格、演示文稿自主工作。月之暗面发布 Kimi K2.5。
- **2026 年 2 月**：Anthropic 发布 **Claude Opus 4.6**，100 万 token 上下文。OpenAI 推出 **GPT-5.3-Codex**，两个版本分别用于长时运行和实时开发。Google 推出 Gemini 3 Deep Think，在 ARC-AGI-2 达到 84.6%。
- **2026 年 3 月**：GPT-5.4 系列发布。**Agent 市场出现第一个"死亡名单"**——Devine 官方披露复杂任务成功率仅 13.86%，比人类慢 10 倍。Manus 用户大量投诉计费异常。
- **2026 年 5 月**：GPT-5.3 系列普及。多 Agent 协作框架（LangGraph、CrewAI、AutoGen）进入生产环境。
- **2026 年 6 月（现在）**：AI Agent 不再是一个"要不要用"的问题，而是"用哪个、怎么用"的问题。

## 二、Agent 到底是什么？先别被名字唬住

很多人以为 Agent 就是"更聪明的 ChatGPT"。

不对。

**Chatbot 和 Agent 的核心区别：**

| | Chatbot | Agent |
|------|------|------|
| 交互模式 | 你问，它答 | 你交代，它去做 |
| 工作方式 | 单次对话 | 多步骤自主执行 |
| 能力边界 | 生成文本 | 操控工具、读写文件、联网、调用 API |
| 记忆 | 无（或极短） | 持久记忆，知道上下文 |
| 类比 | 打字员 | 项目助理 |

简单说：**Chatbot 是在跟你聊天。Agent 是在帮你做事。**

比如你让 Agent "帮我做一个竞品分析报告"：
- 它会自己去搜竞品信息
- 读取你指定的文档
- 整理成表格
- 输出一份结构化的报告

不是给你几条搜索建议。是真的做完。

## 三、当前最火的五个 Agent，逐一说

### 1. Claude Code（Anthropic）

**定位**：开发者日常编程助手

**怎么用**：命令行工具，安装后在终端里直接用自然语言指挥它写代码、改 Bug、读文件、执行命令。

**亮点**：
- 200K 上下文窗口，能吃下中等规模的代码库
- 2025 年更新了 176 次，迭代速度在行业内一骑绝尘
- 写作和代码理解能力在各项评测中稳定前二
- $20/月 Pro 订阅即可用

**槽点**：
- 有每周配额限制，重度使用时会"卡脖子"
- 2025 年曾被发现有远程代码执行漏洞
- 无状态设计——每次新对话都是"从头开始"，不记得上次聊了什么
- "思考 token 压缩"机制偶尔导致回答质量下降

**谁适合**：程序员、技术写作者。日常写代码、调试、重构的性价比之王。

---

### 2. Devin（Cognition Labs）

**定位**：完全自主的 AI 软件工程师

**怎么用**：Web 界面，给它一个需求（比如"搭一个电商网站"），它会自己去规划、写代码、测试、部署。全程不需要你插手。

**亮点**：
- 端到端能力最强——从需求到上线一条龙
- 有完整的 IDE 环境和 Git 集成
- 演示效果惊艳，很多创业公司用来快速出 MVP

**槽点**：
- **成功率只有 13.86%**（2026 年 3 月官方数据）——十次里有八次搞不定
- 平均比人类开发者慢 10 倍
- ACU（Agent Compute Unit）计费模式不透明——复杂任务成本难以预估
- 营销承诺和实际体验差距巨大

**谁适合**：实验性项目、快速原型验证。不建议用于生产环境。你可以把它理解成一个"**非常聪明但极其不靠谱的实习生**"。

---

### 3. Manus（已被 Meta 收购）

**定位**：通用任务 Agent，浏览器自动化

**怎么用**：Web 界面，告诉它要做什么（查资料、填表格、抓数据），它用浏览器模拟你的操作去完成。

**亮点**：
- 浏览器自动化能力强——能真正操控网页，不是只读网页
- 非技术用户友好——不需要懂代码
- 通用性强——从订机票到做调研都能干
- 被 Meta 20 亿美元收购，验证了赛道价值

**槽点**：
- **最大的问题是计费**——大量用户反馈账户被意外耗尽，计费机制像黑洞
- 执行不可靠——经常"自信地做出错误操作"
- GitHub 集成频繁出 Bug，不适合开发场景
- 被收购后路线图不确定，用户有被"抛弃"的风险

**谁适合**：非技术人员的通用任务。但如果要用，建议设消费上限，别让它随便花钱。

**客观评价**：Manus 展示了一个重要的方向——**Agent 不一定非要写代码，操控浏览器也能做很多事**。但它在可靠性上的短板是致命的。被 Meta 收购可能是好事，也可能让它沦为内部工具。

---

### 4. OpenAI Codex / GPT-5.3-Codex

**定位**：OpenAI 官方的编程 Agent

**怎么用**：集成在 ChatGPT 内，直接告诉它编程需求，它会自动规划、写代码、创建 PR。

**亮点**：
- OpenAI 亲儿子，底层模型最新最强
- 2026 年 2 月发布的 GPT-5.3-Codex 有两个版本：长时运行版（Spark）处理复杂工程，实时版快速响应
- 生态系统最完善——插件、GPTs、API，啥都有
- 图像生成独占（DALL-E 系列）

**槽点**：
- $20/月 ChatGPT Plus 是门槛，重度使用还要更多
- 128K 上下文相比 Claude 偏少
- 中文写作偶尔有"翻译腔"，不如 Claude 地道
- 封闭生态——只能用 OpenAI 的东西

**谁适合**：OpenAI 生态用户、需要全能型工具的非开发者。日常写代码 + 做图 + 聊天一个工具搞定。

---

### 5. DeepSeek-R1 / V3

**定位**：国产开源推理模型，性价比之王

**怎么用**：API 调用或 Web 端使用。DeepSeek-R1 专注推理（数学、逻辑、代码），V3 是全功能模型。

**亮点**：
- 推理能力强——数学和逻辑题的表现经常超过 GPT
- **成本极低**——价格是 GPT 系列的五分之一到十分之一
- 开源可用，想自己部署也行
- 中文理解自然，没有"翻译腔"
- R1 在编程推理和调试场景中表现出色

**槽点**：
- 生态和工具链不如 OpenAI 和 Anthropic 完善
- 创意写作能力一般——更适合"逻辑"而不是"文采"
- 偶尔出现"逻辑跳跃"——推理过程中跳步骤
- 没有自己的 Agent 框架（不像 Claude Code 或 Codex 那样能自主操控环境）

**谁适合**：预算敏感的技术用户、中文场景、需要高推理能力的学术和工程任务。

**一句话评价**：如果你的核心需求是"算得对"而不是"写得美"，DeepSeek 是目前最具性价比的选择。

## 四、Agent 框架：幕后的"导演"

除了这些"前台"的产品，还有一个层面值得了解——**Agent 框架**。

如果说 Claude Code、Devin 这些是"演员"，那框架就是"导演"——决定多个 Agent 怎么协作、怎么分工。

### 三大主流框架：

| 框架 | 理念 | 类比 | 最适合 |
|------|------|------|------|
| **LangGraph** | 状态机循环图 | 流水线 + 质检 | 复杂、需要反复修正的企业工作流 |
| **CrewAI** | 角色扮演串联 | 编辑部（记者→编辑→总编） | 内容生产、营销流程、研究报告 |
| **AutoGen** | 对话式辩论 | 会议室（多方激辩达成共识） | 代码审查、安全审计、数学证明 |

**LangGraph** 最灵活但也最难学，适合有经验的团队。**CrewAI** 最简单直观，非技术人员也能上手。**AutoGen** 最像"搭建一个虚拟团队"，但容易陷入无限辩论——需要严格设置终止条件。

一个成熟的 2026 年企业方案往往是**混合架构**：用 AutoGen 做深度研究，把结果交给 CrewAI 做格式化和审核，最后用 Claude Code 完成部署。

## 五、选型指南：到底用哪个？

别被花里胡哨的营销搞晕。回到最实际的问题：**你到底想让它替你干什么？**

| 你的需求 | 推荐 | 月成本 |
|------|------|------|
| 日常写代码、调试 | Claude Code | ~$20 |
| 快速出 MVP | Devin（实验性） | $20+（不可预测） |
| 查资料、填表格、订机票 | Manus（谨慎） | 不可预测 |
| 全能型、做图+聊天+代码 | ChatGPT（GPT-Codex） | $20 |
| 数学推理、学术研究 | DeepSeek-R1 | 极低 |
| 中文写文章、深度内容 | Claude（非 Code） | $20 |
| 不想花钱、先试试 | 豆包 / DeepSeek 免费版 | 免费 |

### 我的实际用法：

我现在的工作流是这样搭的：

1. **Claude Code** 做日常主力——写代码、改 Bug、写文档
2. **DeepSeek-R1** 做数学和逻辑验证——尤其是量化策略的参数计算
3. **ChatGPT** 做图和偶尔的跨语言翻译
4. **所有 Agent 的输出我都要人工过一遍**——这个步骤不能省略，至少目前不能

## 六、几个不讨喜但必须说的真相

### 1. 2026 年的 Agent 依然做不到"替你思考"

Devin 13.86% 的成功率不是偶然。Agent 的本质还是概率机器——它说的一切，都是"最可能对的下一个字"排列出来的。复杂任务的本质不是"排列"，是"判断"。

### 2. 计费透明度是整个行业的耻辱

Manus 的"账户莫名被清空"、Devin 的"ACU 黑洞"、Claude Code 的"配额说没就没"——这些都是真实发生在用户身上的事。在监管跟上来之前，**设消费上限、盯着账单**是必修课。

### 3. 安全漏洞比你想象的严重

Claude Code 在 2025 年被发现可以被恶意代码远程执行。OpenAI 公开承认 Operator 的 Prompt Injection 攻击"目前无法解决"。当你把文件系统权限和网络访问权交给一个概率机器时，它是一个潜在的定时炸弹。

### 4. "Agent 取代程序员"是 2026 年最大的谎言之一

没错，Agent 能加速很多编程工作。但架构设计、业务判断、安全考量、技术选型——这些需要的是"权衡"，不是"计算"。Agent 擅长计算。不擅长权衡。

## 最后

2025 年 1 月，我在 DeepSeek-R1 发布后的第一周就注册了账号。让它做了一套高数题，对了 80%。我当时觉得"这东西会改变一切"。

一年半后的今天回头看，它确实改变了很多——但可能不是你想的那种。

它没有取代程序员。但它让一个普通程序员能干两三个人的活。
它没有消灭低端工作。但它让"会用 AI 的人"和"不会用的人"之间裂开了一道越来越宽的沟。
它不是万能的神。但它是目前我们能找到的最好用的"副驾"。

副驾这个词我说了很多次。但每次说，我都觉得它比上一次说的更贴切。

**能帮你导航。但不能帮你扶方向盘。能提醒你路况。但不能替你踩刹车。**

至于方向盘和刹车——那永远是你自己的事。
