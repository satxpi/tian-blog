十月底，有个叫 Peter 的开发者做了一件挺随意的事——把 WhatsApp 和 Claude Code 连了起来。

不是什么宏大的计划。就是觉得"每次要打开电脑才能用 Claude 写代码太麻烦了，能不能在手机上发条消息就让它干活？"

一个周末写出来的原型。核心逻辑就几十行代码：接收 WhatsApp 消息 → 转发给 Claude API → 把返回结果发回来。

他给这个项目起了个名字：**Clawdbot**。

然后把代码扔到了 GitHub 上。

## 从玩具到 Agent

第一版只能做最简单的事——聊天、翻译、回答问题。

但 Peter 很快发现，Claude Code 在命令行里能干的事，通过 WhatsApp 也能干。而且 Claude 自己就能调用工具——读文件、执行命令、搜索网页。

第二周，他加了文件读写。

第三周，加了本地终端执行。

第四周，加了浏览器自动化。

在社区有人提了个 PR 把 Slack 也接上了。

这时 Clawdbot 已经不是一个"聊天机器人"了——它是你手机上的 AI 助手，能帮你写代码、查资料、改文件、执行脚本。

## 为什么叫"小龙虾"

说实话，这名字挺随意的。

GitHub 上有人问"这 logo 是个钳子吗"，Peter 说"随便画的"。然后评论区就有人管它叫"Claw"（钳子），翻译到中文就成了"小龙虾"。

后来腾讯推广的时候，直接用了"AI 小龙虾"这个名字——比"Clawdbot"亲切一万倍，一下就传开了。

到十二月底，GitHub 上的标星已经过了 5000，Discord 社区进了两千多人。

## 一个有意思的变化

Clawdbot 跟之前的 AI Agent 产品（Manus、Devin）有一个根本不同：

**它不自称"替代你"。**

Manus 说"把你的活交给我，我帮你完成"。
Devin 说"我会写代码，你不用写了"。

Clawdbot 说的是"我就是你的手和脚"——你决定做什么，我帮你操作。

这个定位特别聪明。因为用户不怕 AI 帮干活，用户怕 AI 替自己做决策。

## 我装了一个

折腾了两小时装好（那时候还没一键部署），连上了微信。

第一件让它做的事："查一下我 GitHub 上最近一周的提交记录，汇总成表格。"

十五分钟后它发了个 CSV 文件过来。格式不太对，我让它改了一下。第二版对了。

说实话，那一刻的感觉很奇妙——你躺床上发条消息，AI 在你电脑上吭哧干了半天活，然后把结果发到你手机上。

这不是科幻。这是 2025 年 11 月。
