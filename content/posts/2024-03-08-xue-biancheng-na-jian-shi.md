## 我是怎么入坑的

我不是科班出身。

大学读的是中文系，毕业后进了一家小公司做文案。某天老板叫我"帮忙改一下网站上的字"，我发现那需要登后台改代码，而我看不懂那一堆 `<div>` 和 `{%for%}`。

就是那个下午，我决定学编程。

不是因为热爱（那是后来的事），而是因为**看不懂让我难受**。

---

## 坑1：从 C 语言开始学

第一门语言我选了 C。

原因很蠢：网上说 C 是"最底层的语言，学会了其他语言都不在话下"。

结果我在指针和内存管理里打滚了三个月，一个有实际用处的程序都没写出来，差点放弃。

**建议：** 第一门语言选 Python。没有什么比"写两行代码就能看到效果"更重要。

---

## 坑2：收藏教程替代学习

我收藏了 400 个教程。

我精读了大概 8 个。

**收藏 ≠ 学习。** 打开那个文件夹，选一个，关掉其他所有窗口，动手写代码。

---

## 坑3：跳过报错

初学者对报错有天然恐惧，喜欢重新来一遍，或者粘贴代码"绕过去"。

这是最大的损失。

**报错是最好的老师。** 每一条 `Traceback` 都在告诉你一件具体的事——学会读报错，就是学会了一半的调试能力。

```python
# 这个报错
TypeError: can only concatenate str (not "int") to str

# 告诉你的是：
name = "老田"
age  = 30
print(name + age)       # ❌ 字符串不能直接加整数
print(name + str(age))  # ✅
```

---

## 坑4：学了但不用

我花了两周学了 SQL，然后三个月没有用过。三个月后，全忘了。

**知识是用来提取的，不是用来存储的。** 学完立刻找个场景用，哪怕是造一个假场景。

---

## 坑5：追新技术

2019年：学 Vue 2
2020年：转 Vue 3
2021年：听说 React 更流行，开始学 React
2022年：Next.js 出来了……

我花在"学新框架"上的时间，远多于"用一个框架做出东西"。

**建议：** 先把一个学到能独立做项目，再考虑扩展。

---

## 后来

两年多以后，我能写一个完整的 Web 应用了。

不是因为我比别人聪明，而是因为我终于停止了收藏教程、终于开始读报错、终于忍住了换框架的冲动。

代码这件事，和人生其他大多数事情一样：

> **持续做，比完美计划重要得多。**

---

## 推荐资源

- **Python 入门**：[官方文档](https://docs.python.org/zh-cn/3/) — 质量远高于大多数教程
- **项目驱动**：给自己写一个"自动整理桌面文件"的脚本，比跟着教程敲代码有效10倍
- **遇到问题**：先 Google 英文，再问 AI，再问社区。顺序很重要。
