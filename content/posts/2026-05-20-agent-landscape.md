2026年5月，AI Agent 概念火了整整一年。

去年这个时候，每个人都在说"Agent是AI的iPhone时刻"。一年过去了，该冷静看看：**谁真正在用了，谁还是Demo**。

## 真正在落地的

### Claude Code / Copilot Agent

落地程度：⭐⭐⭐⭐⭐

不需要讨论"要不要用"了。编程圈的共识：**不用 AI Agent 写代码，就是在亏时间。**

使用率从去年的 20% 飙升到今年的 70%+。国外大厂已经把"会用 AI 编程"写进了招聘要求。

为什么落地这么快？因为编程本身是数字化、有标准答案、有即时反馈的——这三个条件完美匹配 Agent 的能力。

### 客服 Agent

落地程度：⭐⭐⭐⭐

银行、电商、运营商的智能客服，去年还被人骂"人工智障"，今年已经能处理 60-70% 的常规问题。

不是技术进步了多少——是**用户习惯了**。大家不再期望跟真人说话，学会用关键词提问，Agent 就能理解。

### 内部运维 Agent

落地程度：⭐⭐⭐

大公司内部 IT 运维、HR 流程、财务审核——这些重复性高、规则明确的工作，Agent 在快速渗透。

一位朋友在字节跳动做技术管理，说他们团队去年 60 人，今年 35 人。"不是裁掉的，是自然的——有人离职了不再补。AI 补上了。"

## 还在画饼的

### "通用"AI Agent

落地程度：⭐（基本是0）

号称"一个 Agent 什么都能做"的产品——没一个真正靠谱的。

原因很简单：通用 = 什么都能做一点 = 什么都做不好。

专业化才是正路。Claude Code 专注于编程，做得比"通用 Agent"好一百倍。

### AI 法律/医疗 Agent

落地程度：⭐

不是技术问题。是**责任问题**。

AI 律师给错了建议，导致你败诉了——谁负责？AI 医生误诊了——谁赔？

在法规明确之前，这些领域 Agent 只能做辅助，不能做决策。

## 落地快的共同特征

我总结了一下，落地快的 Agent 有三个共同点：

1. **有标准答案** —— 代码能跑不能跑，一眼就知道。客服问题有标准SOP。
2. **出错后果可控** —— 代码写错了能改，客服回错了能道歉。法律错了不行。
3. **用户接受度够** —— 程序员接受AI帮写代码，病人不接受AI做诊断。

## 下一个会爆的

我赌**数据分析 Agent**。

数据清洗、可视化、报告生成——规则明确、结果可验证、出错好修正。而且每个公司都需要。

可能会是下一个像 Claude Code 一样疯长的赛道。
