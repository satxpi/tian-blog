## 缘起

去年 ChatGPT 爆火之后，有人开始问：能不能直接让大模型来选股？

我也好奇，花了两个月认真测了一下。结论先说：**LLM 在量化里是有用的工具，但不是万能的选股机器**。

---

## LLM 能做什么

### 1. 财报解读和信息提取

这是目前最稳定的用法。把上市公司的公告、财报 PDF 丢给大模型，让它提取关键数字：

```python
from openai import OpenAI

client = OpenAI()

def parse_annual_report(text: str) -> dict:
    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[
            {"role": "system", "content": "你是一个财务分析师，从财报中提取关键财务数据。"},
            {"role": "user", "content": f"""
从以下财报文本中提取以下字段，以JSON格式返回：
- revenue: 营业收入（亿元）
- net_profit: 净利润（亿元）
- roe: 净资产收益率（%）
- debt_ratio: 资产负债率（%）
- yoy_revenue_growth: 营收同比增速（%）

财报文本：
{text[:3000]}
"""}
        ],
        response_format={"type": "json_object"}
    )
    return json.loads(response.choices[0].message.content)
```

测试了 50 份年报，提取准确率在 90% 左右，主要失误点是复杂表格中的数字对应关系。

### 2. 新闻情感分析

```python
def analyze_sentiment(news_list: list[str]) -> float:
    prompt = f"""
对以下{len(news_list)}条关于某上市公司的新闻进行情感分析。
返回整体情感分数：-1（极度负面）到 1（极度正面）。

新闻标题：
{chr(10).join(news_list)}

只返回一个浮点数，不要解释。
"""
    response = client.chat.completions.create(
        model="gpt-4o-mini",  # 批量处理用便宜模型
        messages=[{"role": "user", "content": prompt}]
    )
    return float(response.choices[0].message.content.strip())
```

这个用法实测 IC 在 0.02-0.03 左右，比随机强，但不是很高。信息优势来自能快速处理大量文本。

### 3. 研报摘要和要点提取

把一份 20 页的研报压缩成 5 个要点，帮你快速浏览。这个不需要代码，直接用，准确率很高。

### 4. 代码辅助

写量化策略时让 LLM 帮你写 Qlib 的配置文件、回测脚本、数据处理代码。这是我用得最多的场景，省了很多查文档的时间。

## LLM 不能做什么

### 不能直接预测涨跌

很多人最大的期望是：把股票数据喂给大模型，让它说"明天涨还是跌"。

这个不行，原因很清楚：

1. **大模型没有实时数据**：训练数据有截止日期，不知道今天的市场行情
2. **价格预测本质上需要统计建模**：LLM 擅长的是语言理解，不是时间序列回归
3. **市场是博弈的**：如果一个模型能稳定预测涨跌，所有人都会用它，然后它就失效了

测试过：把股票的历史 OHLCV 数据格式化成文本，让 GPT-4 预测下一天涨跌。准确率在 50% 附近，跟随机猜没什么区别。

### 不能处理复杂的数值计算

让 LLM 计算一个复杂的 Alpha 因子值，结果往往不准确。LLM 对数学计算有先天局限。

解决方法：让 LLM 写代码，然后用 Python 执行代码。不要让 LLM 直接计算。

### 不能替代对市场的理解

LLM 会给你听起来很有道理的分析，但这些分析可能完全基于训练数据里的常见模式，对具体个股没有真实预测价值。

## 比较靠谱的 LLM 量化工作流

```
市场数据 + 公告文本
        ↓
   LLM 信息提取
   ├── 财报数字提取
   ├── 新闻情感打分
   └── 行业事件分类
        ↓
   结构化特征
        ↓
   传统 ML 模型（LightGBM / XGBoost）
        ↓
   选股信号
        ↓
   风控 + 执行
```

LLM 在这个流程里充当的是**信息处理和特征提取**的角色，而不是直接做预测。最终的量化决策还是靠训练好的 ML 模型。

## ai-hedge-fund：一个有趣的开源项目

GitHub 上有个叫 [ai-hedge-fund](https://github.com/virattt/ai-hedge-fund) 的项目，用多个 LLM Agent 模拟不同的分析师角色（价值投资者、动量分析师、风控官），让他们"开会讨论"后给出买卖建议。

架构很有创意：

```
市场数据
    ↓
┌─────────────────────────────┐
│  价值分析 Agent             │
│  （PE/PB/现金流分析）       │
├─────────────────────────────┤
│  技术分析 Agent             │
│  （趋势/支撑阻力/动量）     │
├─────────────────────────────┤
│  新闻情感 Agent             │
│  （最新新闻/公告/舆情）     │
└─────────────────────────────┘
          ↓（各方观点）
    风控 Agent（综合评估）
          ↓
       交易决策
```

我自己测过：这个框架在确定性强的情况（财报大超预期、政策利好明确）下有参考价值，但对模棱两可的市场环境基本无效，几个 Agent 的分析方向会相互矛盾。

项目本身更多是展示 LLM 多 Agent 架构的工程实现，不是实战级别的策略系统。

## 成本问题

用 LLM 处理大量市场数据，API 成本不可忽视。

以 CSI500 成分股 + 每日新闻情感分析为例：

| 场景 | Token 量/天 | GPT-4o 成本/天 | GPT-4o-mini 成本/天 |
|------|------------|----------------|---------------------|
| 500只股票 × 5条新闻 | 约 25 万 tokens | 约 $1.5 | 约 $0.075 |
| 年报提取（季度） | 约 1000 万 tokens | 约 $60 | 约 $3 |

批量处理优先用 GPT-4o-mini 或者国内的 DeepSeek-V3（价格更低），只有需要高质量理解的场景才用 GPT-4o。

---

## 总结

LLM 在量化里的位置越来越清晰了：

**适合干的**：非结构化信息处理（财报、新闻、研报）、代码辅助、快速原型

**不适合干的**：直接预测价格、替代统计模型、复杂数值计算

最有价值的用法是把 LLM 的语言理解能力和传统量化的统计能力结合起来，各干各擅长的事。期待 LLM 能直接选出 10 倍股，现阶段不太现实。
