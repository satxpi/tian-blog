用了 Claude Code 整整一个月。

不夸张地说——这是我用过**最好的 AI 编程工具**。不是之一。

## Claude Code 跟 Cursor 有什么不一样

Cursor 是"AI 辅助你写代码"。你是主驾驶，AI 帮你踩油门。

Claude Code 是"AI 写代码，你审核"。**AI 是主驾驶，你是副驾驶。**

具体区别：

| | Cursor | Claude Code |
|---|--------|-------------|
| 操作方式 | 你打开文件，AI帮你补全 | 你说要做什么，AI自己操作 |
| 代码范围 | 当前文件 | 整个仓库 |
| 需要人工 | 每行都要 | 只在审核时 |
| 适合 | 写新功能 | 改老项目、重构、Debug |

## 实际场景：重构一个老项目

我把我的博客项目（就是你现在看到的这个网站）丢给了 Claude Code。

指令是："把所有 CSS 变量统一命名规范，把硬编码颜色改成变量。"

它做了这些事：
1. 扫描了整个项目，找到所有 CSS 文件
2. 识别了不一致的命名（`--fg` vs `--text`、`--bg-primary` vs `--bg`）
3. 重命名了所有变量
4. 把所有硬编码颜色值替换成了变量
5. 跑了一遍构建，确认没出错

整个过程一小时。如果我自己做，需要一下午。

更关键的是——**它改的过程中没有引入新 bug**。这是以前的 AI 工具做不到的。

## 最让我惊讶的能力：理解上下文

让 Claude Code 修一个 bug，"首页加载时闪烁了一下"。

它自己打开浏览器开发者工具的截图，找到是 CSS 的 `@keyframes` 在页面首次加载时触发了动画。它改成了 `animation-fill-mode: forwards`。

**它理解了"闪烁"是什么视觉现象，推断出了原因，给出了正确修复。**

这种"看问题 → 推断原因 → 给出修复"的能力，之前的 AI 是完全做不到的。它们只能修"已知问题"——你告诉它第几行有问题，它改那一行。

Claude Code 能修"未知问题"——你描述症状，它自己找原因。

## 不是什么都好

**贵。** API 调用费 + 月费，重度使用每月 $200-300。

**慢。** 复杂任务经常要跑一个多小时。你等着的时候很焦虑。

**会跑偏。** 有一次我让它"优化一下构建速度"，它跑去把整个构建工具从 Webpack 换成了 Vite。没错，确实更快了，但我没让它动架构啊？

## 总结

如果你每天写代码超过 4 小时，Claude Code 值这个钱。

如果你偶尔写写，Cursor 就够了。

但说句实话——用了 Claude Code 一个月，我回不去 Cursor 了。

就像用过自动驾驶的人，回不去手动挡。
