# AWS 计费系统炸了：有人收到了数十亿美元的账单

今天看到一条消息，忍不住要写两句。

AWS 的计费系统出了一个大 bug。有多离谱？**部分用户看到的账单金额是"数十亿美元"级别的。** 对，你没看错，是 billions，单位是"亿"。

## 发生了什么

根据 AWS 官方健康仪表盘的说法，问题出在"预估计费计算子系统中的单位定价计算存在异常"——翻译成人话就是：系统算价格的时候，单位搞错了，只出现在"预估账单"层面，实际扣款不受影响。

但问题是，哪个云用户打开账单看到一串天文数字，第一反应不是心脏骤停？

## 我的几个想法

**第一，云厂商的计费系统从来没"简单"过。**

AWS 有上千种服务、数万种 SKU、复杂的预留实例和节省计划定价模型。计费引擎能跑通已经算是奇迹，出 bug 几乎是必然的。只是这次影响面大了点。

**第二，预估账单和实际账单的差异，是个老大难问题。**

AWS 一直在强调"这只是预估，不是最终费用"。但对于企业财务来说，这种惊吓毫无意义——CFO 看到几十亿美元的"预估"，谁还管你后面说"不是真的"？

**第三，这其实是一个信任问题。**

云厂商最核心的资产不是技术，是信任。你把人家的核心业务跑在你平台上，然后隔三差五发个"几十亿账单"的恶作剧，信任感迟早被消磨干净。

## 尾声

AWS 说用户不需要采取任何行动。但老实说，如果我是那个被多算了数十亿的团队负责人，今天一整天怕是都没心思干别的了。

云原生时代，什么都弹性扩缩，只有心脏承受能力是固定的。

---

这件事也提醒我们：**任何涉及钱的系统，做好可观测性和告警只是基础，关键是别让用户看到吓死人的异常数据而不知道该怎么办。** 这不是技术问题，是产品设计问题。
